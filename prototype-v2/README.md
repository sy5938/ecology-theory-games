# 像森林一样思考 · V3 可玩原型

单人森林演替游戏：玩家只经营一个真实物种，通过持续配置生长、繁殖和储备，在光照、竞争、病原菌、扰动与优势种虫害之间做生命史投资权衡。多物种共存与单种优势都是可能结局。

## 启动与关闭

在仓库根目录运行：

```bash
npm install
npm run demo
```

打开终端显示的本地地址。关闭时回到运行命令的终端，按 `Ctrl+C`。

## V3 可玩内容

- 五步单页向导：选择森林情景、初始密度、生态策略、真实物种，最后确认开局；
- 初始密度可在每 20 m × 20 m 样方 10–400 个体之间调整，对应全图 250–10000 个系统个体；
- 默认密度：密闭森林 30（约 750 个体），稀疏森林 15（约 375 个体），先锋定殖 10（约 250 个系统个体，另加 6 个玩家幼体）；
- 100 m × 100 m 正方形地图，左下角为 `(0,0)`，SVG 绘制清晰标尺、坐标轴和主网格；
- 地图支持滚轮缩放、按住右键拖动平移和一键复位；图层栏采用紧凑宽度，控制条与事件摘要保持固定高度，不因文字变化挤压地图；
- 初始数量确定后，x/y 独立均匀抽样，使用给定数量条件下的齐次泊松点过程（HPP/CSR）；
- 高密度开局以种子、幼苗、幼树和小胸径个体为主，演替中由局部密度和胸高断面积驱动自疏；
- 个体由单个 WebGL 批次中的程序化 SDF 圆绘制，不使用 PNG 或整层树木位图；点径按树高对数映射，并在 overview / stand / individual 三个 GIS 层级间平滑变化；
- 玩家个体默认是无描边的白色矢量五角星，可使用 6 个预设色、颜色轮盘与三档大小自定义，样式保存在 `forest-map-style-v1`；
- 图层树可筛选光照、网格、物种、生活史阶段、林冠/林下、风险、死亡、选中与扰动；点击物种联动统计，双击打开对应属性表；
- 个体点击、多选、玩家幼苗/幼树一次性移栽，以及虚拟滚动属性表和密度/胸径/胸高断面积摘要；
- 统一繁殖成熟阈值为 `min(5 m, 最大树高 × 60%)`，与 10 m 林冠判定分开；
- 先锋型高光快速定殖，耐荫型可低光等待并在林窗中释放生长，广适型保持中等稳定表现；
- 五个快速投资方案，以及诱导防御、集中结实、远距播散、幼苗保育四项主动能力；
- 首次进入有可跳过、可从帮助重播的六步交互教程，覆盖图层、查询、图表、策略、右键平移和演替控制；字体支持小/中/大三档并在本地保存；
- 第 100 年强制结算并解锁长期倍速；玩家物种灭绝后可进入观察模式，整个群落灭绝才是不可继续的终局；
- 统计窗口专注图表，不再重复显示数字指标卡；可选择物种/功能型数量、功能型出生死亡个体、功能型出生死亡率、阶段、断面积、多样性、固碳与选中对象；
- 每年记录物种、功能型与群落的出生、死亡、标准化发生率和固碳；ZIP 导出包含 `species_history.csv`、`functional_type_history.csv` 与 `community_history.csv`；
- 普通个体死亡不逐条刷屏，突发事件即时显示，无突发事件时每五年汇总冠层优势种和玩家种群变化；
- 默认工作台采用“左侧地图、右上统计图、右下生命史策略”，地图窗格向右扩展约 35 px，统计区默认占右栏约 62%；“放大查看”仅纵向占据右栏，不遮挡地图；窗口可自由拖动、缩放、最小化和最大化，拖动后不自动吸附或归位，状态保存在 `forest-workspace-layout:v7`；窄屏使用地图加底部抽屉；
- 10,000 个体时保持 WebGL 批绘和空间索引命中，光照仍作为连续环境栅格缓存；非关键统计降低刷新频率，16× 使用分帧步进避免阻塞交互。

## 验证命令

```bash
npm run test:model
npm run test:browser
npm run build:demo
```

模型测试使用固定随机种子检查密度换算、HPP/CSR、径级与自疏、策略差异、成熟阈值、固碳加和不变量、高密度 16× 和 ZIP；浏览器测试使用本机 Chrome 检查向导、教程、字体、SVG 标尺、GIS 图层、玩家样式、统计联动、可调工作台、移动抽屉、虚拟滚动，以及 10,000 个体至少 30 FPS 的交互。
